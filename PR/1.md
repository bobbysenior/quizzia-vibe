# PR #1 — Mise en place du projet

## Résumé

Initialisation de l'application Next.js (App Router, TypeScript, Tailwind v4) et du schéma de base de données Supabase pour l'app Quizz.

---

## Changements

### Base de données

- **Migration SQL** (`migrations/20260516000000_init.sql`) — schéma complet :
  - 5 tables : `quizzes`, `questions`, `choices`, `quiz_attempts`, `user_answers`
  - 6 index dont 1 partiel (`WHERE status = 'published'`)
  - 2 triggers (`updated_at` automatique, contrainte 1 réponse correcte par question)
  - RLS activé sur les 5 tables avec 15 policies couvrant US-01 à US-18

### Frontend — Next.js

- **App Next.js 16** avec TypeScript, Tailwind v4, Turbopack
- **Dépendances** : `@supabase/ssr`, `@supabase/supabase-js`, `@tanstack/react-query`, `zod`
- **Landing page** minimaliste (style Apple) :
  - Section héro avec titre gradient, sous-titre, double CTA
  - 3 cartes fonctionnalités (Parcourir / Créer avec IA / Progresser)
  - Footer léger
- **Clients Supabase** :
  - `src/lib/supabase/client.ts` — client navigateur
  - `src/lib/supabase/server.ts` — client serveur (cookies)

### Configuration

- `.env.local` — variables Supabase (URL + clé anon)
- `tsconfig.json` — exclusion de `volumes/`, `tests/`, `utils/`, `dev/`, `migrations/`, `supabase/`
- `.gitignore` — ajout de `.next`, `.env.local`, `.commandcode`, `supabase/.temp`, `*.tsbuildinfo`
- Correction Turbopack : directive `@source "../../src"` dans `globals.css`
