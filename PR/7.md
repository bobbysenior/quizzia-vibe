# PR #7 — Génération de quiz par IA (OpenAI)

## Résumé

Implémentation de la génération de quiz par intelligence artificielle. Un utilisateur décrit le quiz souhaité, le backend appelle OpenAI qui génère un JSON structuré, puis insère quiz + questions + choix en base.

---

## Fichiers créés

| Fichier | Rôle |
|---|---|
| `src/lib/ai/openai.ts` | Client OpenAI singleton |
| `src/lib/ai/generate-quiz.ts` | Schéma Zod du JSON de sortie, system prompt, appel `gpt-4o` avec `response_format: json_object` |
| `src/lib/ai/create-quiz-from-llm.ts` | Insertion batch en BDD (quiz → questions → choix) avec la service role key |
| `src/app/api/quizzes/generate/route.ts` | Endpoint `POST /api/quizzes/generate` |

## Fichiers modifiés

| Fichier | Changement |
|---|---|
| `package.json` | Ajout de `openai` |
| `.env.example` | Ajout `OPENAI_API_KEY` + `OPENAI_MODEL` |
| `src/app/quizzes/new/prompt-form.tsx` | Branchement sur l'API : champs min/max questions, appel fetch avec Bearer token, redirection vers l'éditeur |

## Format JSON attendu du LLM

```json
{
  "title": "Titre du quiz",
  "theme": "Thème",
  "questions": [
    {
      "question_text": "Question ?",
      "choices": [
        { "choice_text": "Option A", "is_correct": false },
        { "choice_text": "Option B", "is_correct": true }
      ]
    }
  ]
}
```

## Endpoint API

```
POST /api/quizzes/generate
Authorization: Bearer <access_token>
Content-Type: application/json

{
  "prompt": "Un quiz sur le cinéma français",
  "min_questions": 5,
  "max_questions": 10
}

→ 201 { "quizId": "uuid" }
```

## Variable d'environnement à ajouter

```
OPENAI_API_KEY=sk-...
# OPENAI_MODEL=gpt-4o (optionnel)
```

## Flow

1. Utilisateur remplit le prompt + min/max → soumet
2. `PromptForm` récupère l'access token Supabase → `POST /api/quizzes/generate`
3. Le endpoint vérifie le token, valide le body, appelle OpenAI
4. Le LLM répond en JSON → validation Zod → insertion BDD
5. Redirection vers `/quizzes/:id/edit` pour édition manuelle
