# Plan : Limiter à 4 choix, suppression fonctionnelle, sauvegarde groupée, bouton Enregistrer visible

## Problèmes actuels

1. **Choix illimités** — `addChoice` permet d'ajouter indéfiniment
2. **Suppression de choix cassée** — `deleteChoiceAction` supprime côté serveur mais `onRemove` n'est pas appelé localement. Pour les choix sans `id`, le bouton ✕ n'apparaît même pas.
3. **Sauvegarde individuelle des choix** — chaque choix a son propre `<form action={cAction}>`, donc sauvegardé séparément
4. **Bouton Enregistrer discret** — petit lien textuel "Enregistrer la question" en bas du formulaire

## Solution

Toutes les modifications (texte question, texte choix, ordre, correct/incorrect, ajout/suppression) sont **locales** dans le state React. La persistance DB se fait au clic sur **"Enregistrer"**, qui sauvegarde la question ET tous ses choix.

```
Édition locale → [Enregistrer] → saveQuestionAndChoicesAction → upsertQuestion + upsertChoice × N
```

## Fichiers modifiés

### 1. `src/lib/quiz/actions.ts`

Nouvelle action `saveQuestionAndChoicesAction` :
- Reçoit `quizId` (bind), `_prevState`, `formData`
- Parse les champs : `id`, `question_text`, `order_index`, `choices` (JSON)
- Appelle `upsertQuestion` → récupère l'ID
- Pour chaque choix non vide : appelle `upsertChoice`
- Retourne `{ success, questionId }` ou `{ error }`

### 2. `src/app/quizzes/[id]/edit/quiz-editor.tsx`

**QuestionEditor :**
- `useActionState(saveQuestionAndChoicesAction.bind(null, quizId), null)`
- Bouton "Enregistrer" dans l'en-tête à côté de "Supprimer", style `bg-ink text-white` (visible, primaire)
- `<form id={`question-form-${qIdx}`}>` avec `<input type="hidden" name="choices" value={JSON.stringify(question.choices)} />`
- Limite "+ Ajouter un choix" → `question.choices.length < 4`

**ChoiceRow (simplifié) :**
- Retrait de tous les `useActionState` (saveChoice, deleteChoice, setCorrectChoice)
- Retrait de tous les `<form action={...}>`
- `onSetCorrect` → appel local uniquement
- `onRemove` (✕) → appel local uniquement, toujours visible
- `onTextChange` → local uniquement

## Vérification

1. Ajouter des choix → limité à 4 (bouton "+" disparaît)
2. Supprimer un choix → disparaît immédiatement du DOM
3. Modifier les textes → local uniquement, pas de requête serveur
4. Cliquer "Enregistrer" → question + choix persistés
5. Recharger la page → données sauvegardées
6. Bouton Enregistrer visible dans l'en-tête
