## Script de seed (`npm run seed`)

Le script utilise la `SERVICE_ROLE_KEY` (clé admin Supabase) pour insérer des données sans restriction RLS.

### Données générées

- **1 compte démo** : `demo@quizz.app` / `demo123456`
- **3 quizz publiés** :
  - Les mystères de l'univers (Science)
  - Les grandes dates de l'Histoire (Histoire)
  - Capitales du monde (Géographie)
- **5 questions par quizz** avec 4 choix chacune (1 correct)
- **1 tentative complétée** pour le compte démo (score 4/5)

### Fonctionnement

1. Lit `.env` pour `SUPABASE_PUBLIC_URL` et `SERVICE_ROLE_KEY`
2. Crée le compte démo via Admin API
3. Supprime les anciens quizz de seed (idempotent)
4. Insère quizz → questions → choix
5. Crée une tentative complétée avec réponses

### Dépendances

Aucune dépendance externe — utilise `@supabase/supabase-js` (déjà installé) et les modules natifs Node.js (`fs`, `path`, `url`).
